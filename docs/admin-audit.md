## Method

Source of truth = the cumulative schema after migrations `001`–`031`. For every
admin route the audit extracted each `.from(...)`, `.select(...)`, `.order(...)`,
`.eq/.gte/.lte(...)`, `.insert(...)`, `.update(...)` and `.rpc(...)` call and
checked each referenced table, column, embed target and FK path against that
schema. Both `npx tsc --noEmit` (client + server) are clean at audit time.

### Schema reference (columns actually present on the hot tables)

- **accounts** — `id, user_id, type, status, currency, balance, equity,
  margin_used, free_margin, leverage, affiliate_id, created_at, updated_at`,
  `login` (003), `hedging_enabled` (022), `is_primary` (024), `last_seen` (031).
- **trades** — `id, account_id, symbol, side, volume, open_price, current_price,
  close_price, stop_loss, take_profit, swaps, commission, profit, reason,
  comment, robot_id, status, open_time, close_time`, plus `client_request_id`
  (015), `order_type`/`trigger_price` (016), `limit_price` (018),
  `trail_distance`/`trail_high_water` (019), `oco_group_id` (020), `notes` (023).
  **Time columns are `open_time` / `close_time`** (not `opened_at`/`closed_at`).
- **transactions** — `id, account_id, type, amount, currency, status, method,
  reference, notes, created_at, completed_at`. (No `reviewed_at`/`updated_at`.)
- **profiles** — `id` (= auth uid), `display_name, phone, country, ui_mode,
  online_status, last_seen_at, push_token, notification_prefs, created_at,
  updated_at`, plus `current_streak`/`best_streak` (005), `is_admin` (007),
  `last_login_date`/`login_streak` (011), `copy_leader_enabled` (024),
  `share_trades` (027), `risk_accepted_at` (028). **No direct FK to accounts**
  (both reference `auth.users`) — must stitch by `user_id`.
- **kyc_submissions** — `id, user_id, status, rejection_reason, submitted_at,
  reviewed_at, created_at`. **kyc_documents** — `id, submission_id, doc_type,
  storage_path, uploaded_at` (FK `submission_id`).
- **robots** — `id, account_id, name, …`. **robot_runs** — `id, robot_id,
  triggered_at, action, payload, trade_id, notes` (no FK to accounts; two-hop
  via robots→accounts).
- **price_alerts** — `id, user_id, symbol, threshold, direction, triggered_at,
  created_at` (no `status` column — derived from `triggered_at`; no `account_id`).
- **notifications** — `id, user_id, kind, title, body, symbol, data, read_at,
  created_at`.
- RPC `apply_trade_pnl(p_account_id uuid, p_amount numeric)` defined in `002`.

## Findings

1. **No column-name bug found.** The two historically-fixed defects — `opened_at`
   → `open_time` (`/risk`, 2026-06-11) and the invalid profiles↔accounts
   PostgREST embed → stitch-by-`user_id` (`/users`, commit `0d4d991`) — are both
   confirmed corrected and have not regressed. Every other route already uses the
   live column names and stitches the FK-less profiles/accounts/robot_runs
   relations in application code rather than via embeds.

2. **`GET /online` carries a live dependency, not a code bug.** It reads
   `accounts.last_seen`, added by `031_account_last_seen.sql`, which per STATE.md
   is **not yet applied to the live DB**. Until applied, this route will return
   500 (`query_failed`) on live even though the code is correct. Apply on a
   network-enabled run:
   `SUPABASE_PAT=... python scripts/apply-migration.py supabase/migrations/031_account_last_seen.sql`

3. **Storage vs table.** `GET /kyc` uses `supabaseAdmin.storage.from('kyc')` to
   mint 1-hour signed URLs — that `'kyc'` is a **Storage bucket name**, not a
   missing table. Correct as written.
